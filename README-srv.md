# AutoTestSrv 自动化测试服务端

## 项目概述

**AutoTestSrv** 是 AutoTest 自动化测试平台的服务端应用，基于 Android 提供 HTTP API 服务，支持设备远程管理和实时控制。

### 核心功能

- **HTTP API 服务**：提供 RESTful API 接口进行用户认证和设备管理
- **用户管理**：支持用户注册、登录、JWT Token 认证
- **设备管理**：设备绑定、在线状态监控、分页查询

## 使用指南

### 安装应用

1. 将 APK 文件安装到 Android 设备
2. 首次启动需要授予必要的权限
3. 服务启动后会自动运行 HTTP 服务器

### 服务配置

服务默认运行端口：
- **HTTP 服务**：`http://<手机IP>:8080`

### 功能模块说明

#### 1. HTTP API 服务
- 用户注册和登录
- 设备列表查询
- 服务状态监控

## API 接口文档

### 基础信息

| 项目 | 说明 |
|------|------|
| 基础地址 | `http://<设备IP>:8080` |
| 数据格式 | JSON |
| 认证方式 | JWT Token (Bearer Token) |

### 公共响应格式

```json
{
  "success": true|false,
  "message": "响应消息",
  "data": { ... }
}
```

### 接口列表

#### 1. 状态接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/status` | GET | 获取服务状态 |

**响应示例：**
```json
{
  "success": true,
  "data": {
    "status": "running",
    "httpPort": 8080,
    "signalingPort": 8765
  }
}
```

## 常见问题

### Q1: 无法连接到 HTTP 服务？
- 检查应用是否已授予网络权限
- 确认防火墙未拦截端口 8080
- 查看设备 IP 地址是否正确

### Q2: 设备显示不在线？
- 确认设备端已连接到信令服务器
- 检查设备端注册是否成功
- 查看设备和控制端是否属于同一用户
